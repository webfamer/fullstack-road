# AI 项目指标核实清单

以下数字是同类内部 AI 应用的常见参考区间，不代表你的真实成果，不能未经核实直接写进简历。

## 企业知识库管理平台

- 文档规模：`1,000-10,000 份`或`10 万-100 万 chunks`
- 内部用户：`20-200 人`
- 人工评测集：`50-200 个典型问题`
- Top-5 召回命中率：`70%-90%`
- Rerank 相比纯向量召回提升：`5-15 个百分点`
- 首 Token 延迟：`1-3 秒`
- 完整回答耗时：`5-15 秒`
- 文档入库耗时：按`单份/百页/千份批量`分别核实

建议从数据库、对象存储、向量表、访问日志和评测脚本中反查真实数据。

## BI 智能问数 Agent

- 覆盖数据表：`20-100 张`
- 指标口径：`50-300 个`
- 典型问题测试集：`50-200 条`
- SQL 首次生成可执行率：`70%-90%`
- 经校验/修正后的可执行率：`85%-97%`
- 简单查询响应：`3-8 秒`
- 复杂查询响应：`8-20 秒`
- 高风险 SQL 拦截：应通过构造测试集统计，不要凭感觉填写

建议从测试用例、SQL 执行日志、LangGraph 节点状态和业务元数据中反查真实数据。

## BI 报表分析平台

- 组件库复用项目数
- 图表/表格组件数量
- 首屏加载时间改造前后值
- 静态资源体积或请求数变化
- 线上问题平均定位时间

## 可安全使用的降级表达

如果拿不到数字，可写成：

- 建立从文件上传到索引入库的完整自动化流程
- 从单一向量召回升级为关键词与向量混合召回
- 为生成 SQL 增加执行前安全校验与错误修正
- 将结果从原始数据升级为表格和图表可视化呈现
- 建立 RAG 全链路日志，使召回和生成问题可追踪
