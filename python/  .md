## 1. 先建立心智模型：Python 和 JavaScript 最大的不同

如果你是从 JS 切过来，先记这几条：

1. **代码块靠缩进，不靠 `{}`**
2. **变量默认就是绑定，不需要 `let`/`const`**
3. **`dict` 很像对象，但它本质是字典，不是基于原型链的对象系统**
4. **循环更常见的是 `for item in items`，不是传统 `for (let i = 0; ...)`**
5. **类能写，但很多场景 Python 更鼓励直接写函数、字典、数据类**
6. **标准库很好用，很多事先别急着装包**

一个最直观的对比：

```javascript
if (score > 60) {
  console.log('pass')
} else {
  console.log('fail')
}
```

```python
if score > 60:
    print('pass')
else:
    print('fail')
```

你会发现：

- 没有括号也能写 `if`
- 没有大括号
- 代码块靠缩进
- 行尾通常不用分号

## 2. 环境准备：先会跑 Python

查看版本：

```bash
python3 --version
```

创建虚拟环境：

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
```

退出虚拟环境：

```bash
deactivate
```

为什么要这样做？因为官方教程把 `venv` 作为隔离项目依赖的标准做法；同一个机器上不同项目可以有不同包版本。官方教程也明确给出创建方式：`python -m venv tutorial-env`。  

如果你熟悉 Node，这个感觉有点像：

- `nvm` / 不同 Node 版本 + 项目自己的依赖目录
- 但 Python 更强调“每个项目一个虚拟环境”

## 3. 变量、基本类型、打印

### JavaScript 对照

```javascript
const name = 'Ada'
let age = 18
const active = true
console.log(`${name} is ${age}`)
```

### Python 写法

```python
name = 'Ada'
age = 18
active = True
print(f'{name} is {age}')
```

注意点：

- `true/false/null` 在 Python 里是 `True/False/None`
- 字符串插值优先用 **f-string**
- Python 是动态类型，但你可以加类型标注帮助阅读和 IDE

```python
def greet(name: str) -> str:
    return f'Hello, {name}'
```

## 4. 最常用容器：list、dict、tuple、set

### 4.1 list ≈ JS Array

```python
numbers = [1, 2, 3]
numbers.append(4)
print(numbers[0])
print(numbers[-1])
```

### 4.2 dict ≈ JS Object，但别完全等同

```python
user = {
    'name': 'Ada',
    'age': 18,
    'skills': ['Python', 'JavaScript'],
}

print(user['name'])
print(user.get('email'))
print(user.get('email', 'no-email'))
```

和 JS 不同的感受：

- Python `dict` 就是字典结构，不是“对象 + 原型链”
- 访问不存在的 key，`user['email']` 会报错；更稳的是 `get()`

### 4.3 tuple：不可变序列

```python
point = (10, 20)
```

适合“就是一组固定值，不想被改”。

### 4.4 set：去重和集合运算很好用

```python
skills = {'js', 'python', 'js'}
print(skills)  # {'js', 'python'}
```

## 5. 一个非常 Python 的特性：推导式

如果你是 JS 开发者，可以把它先理解成 `map + filter` 的紧凑写法。

### JavaScript

```javascript
const numbers = [1, 2, 3, 4, 5, 6]
const evenSquares = numbers.filter(n => n % 2 === 0).map(n => n * n)
```

### Python

```python
numbers = [1, 2, 3, 4, 5, 6]
even_squares = [n * n for n in numbers if n % 2 == 0]
print(even_squares)
```

Python 官方教程把 list comprehension 视为创建列表的简洁方式，很值得优先掌握。

再看一个：

```python
names = ['ada', 'grace', 'linus']
upper_names = [name.upper() for name in names]
print(upper_names)
```

别一上来就写太复杂的嵌套推导式；能读懂比“炫技”重要。

## 6. 条件、循环、真假值

### 6.1 if

```python
score = 75
if score >= 90:
    print('A')
elif score >= 60:
    print('B')
else:
    print('C')
```

### 6.2 for：更像 `for...of`

```python
skills = ['js', 'python', 'sql']
for skill in skills:
    print(skill)
```

如果你又想拿索引又想拿值：

```python
for index, skill in enumerate(skills, start=1):
    print(index, skill)
```

### 6.3 while

```python
count = 3
while count > 0:
    print(count)
    count -= 1
```

### 6.4 truthy / falsy

下面这些通常会被当成假：

```python
False
None
0
''
[]
{}
set()
```

所以很常见的写法是：

```python
items = []
if not items:
    print('empty')
```

## 7. 函数：比 JS 更常见关键字参数

### 基础写法

```python
def add(a, b):
    return a + b
```

### 默认参数

```python
def greet(name, prefix='Hi'):
    return f'{prefix}, {name}'
```

### 关键字参数

```python
print(greet('Ada', prefix='Hello'))
```

这点很像 JS 里传配置对象，但 Python 更直接。

### 可变参数

```python
def total(*numbers):
    return sum(numbers)

print(total(1, 2, 3))
```

```python
def print_user(**kwargs):
    print(kwargs)

print_user(name='Ada', age=18)
```

### 一个新手高频坑：可变默认参数

别这样写：

```python
def append_item(item, items=[]):
    items.append(item)
    return items
```

应改成：

```python
def append_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items
```

这个坑和 JS 的默认参数体验不太一样，值得提前记住。

## 8. 类：先会基本类，再学 `@dataclass`

你的参考材料里已经有一篇 `python/深入理解类.md`，那篇更偏类本身。  
这里我给 JS 开发者一个“先够用”的版本。

### 8.1 普通类

```python
class User:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def greet(self):
        return f'Hi, I am {self.name}'

user = User('Ada', 18)
print(user.greet())
```

对 JS 开发者来说：

- `__init__` 很像 constructor
- `self` 类似 JS 里的 `this`，但要显式写出来
- 实例属性常在 `__init__` 里赋值

### 8.2 更推荐的入门姿势：`@dataclass`

如果你的类主要是“存数据 + 少量方法”，优先考虑数据类：

```python
from dataclasses import dataclass

@dataclass
class Task:
    title: str
    done: bool = False

    def toggle(self):
        self.done = not self.done
```

`@dataclass` 会帮你自动生成常见方法，比如 `__init__()`、`__repr__()`，非常适合入门时减少样板代码。

## 9. 模块与 import：比 JS 朴素，但很好懂

假设有一个文件 `math_tools.py`：

```python
def add(a, b):
    return a + b
```

另一个文件里这样用：

```python
from math_tools import add

print(add(1, 2))
```

先记住三件事：

1. 一个 `.py` 文件就是一个模块
2. `import` 会导入模块
3. 代码不要全堆一个文件，学会拆模块很重要

## 10. 文件读写：现在优先学 `pathlib`

老教程常见 `os.path`，但现代 Python 里，官方文档对 `pathlib` 的定位非常明确：如果你不确定该用哪个路径类，`Path` 通常就是你需要的。

```python
from pathlib import Path

file_path = Path('notes.txt')
file_path.write_text('hello python', encoding='utf-8')
content = file_path.read_text(encoding='utf-8')
print(content)
```

这比手写字符串拼路径舒服很多，也更清晰。

## 11. 异常处理：别害怕 `try/except`

### JavaScript

```javascript
try {
  const n = Number.parseInt(value)
  if (Number.isNaN(n)) throw new Error('bad number')
} catch (err) {
  console.error(err)
}
```

### Python

```python
try:
    n = int(value)
except ValueError:
    print('bad number')
```

Python 的风格很强调：

- 先做
- 出错再捕获

也就是常说的 EAFP（先执行，异常再处理）风格。入门阶段先记住：**捕获具体异常，不要上来就 `except Exception`。**

## 12. 一个小而完整的例子

下面这个例子把列表、字典、函数、推导式都串起来：

```python
def build_report(orders):
    paid_orders = [order for order in orders if order['paid']]
    total = sum(order['amount'] for order in paid_orders)
    names = [order['name'] for order in paid_orders]
    return {
        'count': len(paid_orders),
        'total': total,
        'names': names,
    }

orders = [
    {'name': 'book', 'amount': 30, 'paid': True},
    {'name': 'pen', 'amount': 10, 'paid': False},
    {'name': 'bag', 'amount': 80, 'paid': True},
]

print(build_report(orders))
```

如果用 JS 思维去理解，就是：

- `filter` 出已支付订单
- `reduce`/`sum` 计算总价
- `map` 提取名称
- 最后返回一个对象（Python 里是 `dict`）

## 13. 给 JS 开发者的迁移建议

### 建议 1：先别急着找“Python 版类组件式架构”

Python 完全支持类，但你会发现很多逻辑用：

- 函数
- 模块
- 字典
- 数据类

就够了。

### 建议 2：多写“Pythonic”代码

例如：

```python
if not users:
    print('no users')
```

比下面更自然：

```python
if len(users) == 0:
    print('no users')
```

### 建议 3：遇到“遍历 + 转换 + 过滤”，先想推导式

```python
active_names = [user['name'] for user in users if user['active']]
```

### 建议 4：文件路径优先 `pathlib`

### 建议 5：数据对象优先考虑 `dataclass`
