## 1. JavaScript 必会

### 数据类型与类型判断

必会问题：

- JS 有哪些数据类型？
- `typeof null` 为什么是 `object`？
- `typeof`、`instanceof`、`Object.prototype.toString.call` 区别？
- `==` 和 `===` 区别，隐式转换规则？
- `Object.is` 和 `===` 区别？
- `0.1 + 0.2 !== 0.3` 原因和解决方式？

答题关键词：

- 基本类型：`string`、`number`、`boolean`、`undefined`、`null`、`symbol`、`bigint`。
- 引用类型：对象、数组、函数、日期、正则等。
- `typeof` 适合判断基本类型和函数，但不能细分对象。
- `instanceof` 基于原型链，跨 iframe 或手动改原型时可能不准。
- `Object.prototype.toString.call` 最稳，可区分数组、日期、正则等。
- 浮点数问题来自二进制表示和 IEEE 754 精度限制，可用误差范围、整数化、Decimal 库处理。

### 原型、原型链、继承

必会问题：

- 什么是原型和原型链？
- 构造函数、实例、`prototype`、`__proto__` 的关系？
- `new` 做了什么？
- ES5/ES6 继承方式有哪些？

答题骨架：

> 每个函数都有 `prototype`，实例对象通过内部原型指向构造函数的 `prototype`。访问属性时，如果对象自身没有，会沿着原型链向上查找，直到 `Object.prototype` 或 `null`。

`new` 的过程：

1. 创建空对象。
2. 将对象原型指向构造函数的 `prototype`。
3. 执行构造函数，`this` 指向新对象。
4. 如果构造函数返回对象，则返回该对象，否则返回新对象。

### 作用域、闭包、执行上下文

必会问题：

- 作用域链是什么？
- 闭包是什么，有什么应用和风险？
- 执行上下文创建过程？
- 变量提升是什么？

答题关键词：

- 词法作用域：函数作用域在定义时决定。
- 执行上下文：变量环境、词法环境、`this` 绑定。
- 闭包：函数可以访问其定义时外层作用域的变量，即使外层函数已经执行完。
- 应用：私有变量、函数柯里化、防抖节流、模块化。
- 风险：长期引用外部变量可能导致内存无法释放。

### this、call、apply、bind

必会问题：

- `this` 指向规则？
- 箭头函数和普通函数区别？
- `call/apply/bind` 区别？

答题关键词：

- 默认绑定：非严格模式指向 `window`，严格模式是 `undefined`。
- 隐式绑定：谁调用指向谁。
- 显式绑定：`call/apply/bind` 指定。
- `new` 绑定优先级高。
- 箭头函数没有自己的 `this`，继承外层词法作用域 `this`，不能作为构造函数。

### 异步与事件循环

必会问题：

- 事件循环机制？
- 宏任务和微任务有哪些？
- `Promise`、`async/await`、`setTimeout` 执行顺序？
- `Promise.all`、`race`、`allSettled` 区别？

答题骨架：

> JS 主线程先执行同步代码，遇到异步任务交给宿主环境。同步代码执行完后，清空微任务队列，再取一个宏任务执行。每执行完一个宏任务，都会清空当前产生的微任务。

常见分类：

- 宏任务：`script`、`setTimeout`、`setInterval`、I/O、UI 渲染、MessageChannel。
- 微任务：`Promise.then/catch/finally`、`queueMicrotask`、`MutationObserver`。
- Node 里还要注意 `process.nextTick` 优先级更高。

### 垃圾回收与内存泄漏

必会问题：

- V8 垃圾回收机制？
- 哪些操作会造成内存泄漏？

答题关键词：

- 常见回收算法：标记清除、引用计数、分代回收。
- 新生代适合频繁创建销毁对象，老生代存活时间更长。
- 内存泄漏场景：全局变量、未清理定时器、未解绑事件、闭包长期持有、大量 DOM 引用、缓存无限增长。

## 3. 浏览器必会

### 输入 URL 到页面展示

必会流程：

1. URL 解析。
2. 查缓存。
3. DNS 解析。
4. TCP 连接，HTTPS 还要 TLS 握手。
5. 发送 HTTP 请求。
6. 服务端响应。
7. 浏览器解析 HTML，构建 DOM。
8. 解析 CSS，构建 CSSOM。
9. 合成 Render Tree。
10. Layout、Paint、Composite。
11. JS 可能阻塞解析和渲染。

追问点：

- DNS 缓存位置。
- HTTPS 握手过程。
- CSS 和 JS 如何阻塞。
- 首屏优化可以从哪些阶段切入。

### 渲染机制

必会问题：

- DOM、CSSOM、Render Tree、Layout、Paint、Composite 区别？
- 回流和重绘区别？
- 如何减少回流重绘？
- 为什么动画推荐 `transform`、`opacity`？

答题关键词：

- 回流影响几何信息，成本更高；回流一定触发重绘。
- 重绘只改颜色、背景、阴影等视觉样式。
- 避免频繁读写布局属性交叉执行。
- 批量 DOM 操作、使用 class、脱离文档流、`requestAnimationFrame`。
- `transform/opacity` 通常可以走合成层，减少 layout/paint。

### 浏览器缓存

必会问题：

- 强缓存和协商缓存？
- `Cache-Control`、`Expires`、`ETag`、`Last-Modified`？
- 地址栏回车、F5、强刷区别？

答题骨架：

> 强缓存命中时不发请求，主要看 `Cache-Control` 和 `Expires`。强缓存失效后进入协商缓存，浏览器带上 `If-None-Match` 或 `If-Modified-Since`，服务端判断资源没变则返回 304。

关键词：

- `Cache-Control: max-age` 优先级高于 `Expires`。
- `ETag` 优先级高于 `Last-Modified`，更精确但服务端计算有成本。
- HTML 通常不强缓存或短缓存，带 hash 的静态资源可以长缓存。

### 本地存储

必会问题：

- Cookie、localStorage、sessionStorage 区别？
- IndexedDB 用在什么场景？
- Cookie 常见属性？

关键词：

- Cookie 会随请求发送，容量小，适合认证相关。
- localStorage 持久化，容量较大，同源共享。
- sessionStorage 标签页级别，关闭即释放。
- Cookie 属性：`HttpOnly`、`Secure`、`SameSite`、`Domain`、`Path`、`Max-Age/Expires`。
- IndexedDB 适合大量结构化离线数据。

### 安全

必会问题：

- XSS 是什么，怎么防？
- CSRF 是什么，怎么防？
- CSP、HttpOnly、SameSite 分别解决什么？

答题关键词：

- XSS：恶意脚本注入。防御：输入校验、输出转义、富文本白名单、CSP、HttpOnly。
- CSRF：借用户登录态发起伪造请求。防御：SameSite Cookie、CSRF Token、Referer/Origin 校验、关键操作二次验证。

## 4. 网络必会

### HTTP 基础

必会问题：

- GET 和 POST 区别？
- HTTP 状态码？
- HTTP/1.1、HTTP/2、HTTP/3 区别？
- keep-alive 是什么？
- OPTIONS 预检请求是什么？

答题关键词：

- GET 偏获取，参数常放 URL；POST 偏提交，参数常放 body。语义和幂等性比“能不能传 body”更重要。
- 200、204、301、302、304、400、401、403、404、405、409、500、502、503、504。
- HTTP/1.1：长连接、管线化但有队头阻塞。
- HTTP/2：二进制分帧、多路复用、头部压缩、服务端推送。
- HTTP/3：基于 QUIC/UDP，改善 TCP 队头阻塞和连接迁移。
- OPTIONS 预检：复杂跨域请求前，浏览器先确认服务端是否允许方法、头和来源。

### HTTPS

必会问题：

- HTTPS 如何保证安全？
- TLS 握手过程？
- 证书解决什么问题？

答题骨架：

> HTTPS = HTTP + TLS。TLS 通过非对称加密协商会话密钥，再用对称加密传输数据；证书用于证明服务端身份，防止中间人伪造。

关键词：

- 身份认证、机密性、完整性。
- CA、证书链、公钥、私钥、会话密钥。

### DNS、TCP、UDP

必会问题：

- DNS 查询过程？
- TCP 三次握手、四次挥手？
- TCP 如何保证可靠？
- TCP 和 UDP 区别？

关键词：

- DNS：浏览器缓存、系统缓存、hosts、本地 DNS、根域名、顶级域名、权威域名。
- TCP：面向连接、可靠、有序、流量控制、拥塞控制、重传。
- UDP：无连接、低延迟、适合直播、音视频、DNS、QUIC。

## 5. CSS 必会

### 布局基础

必会问题：

- 标准盒模型和 IE 盒模型？
- BFC 是什么，怎么触发，有什么用？
- Flex 常见属性？
- 水平垂直居中方案？
- 两栏/三栏布局？

答题关键词：

- `box-sizing: content-box/border-box`。
- BFC 是独立布局上下文，可解决 margin 重叠、清除浮动、阻止文字环绕浮动。
- BFC 触发：`overflow` 非 visible、`display: flow-root/flex/grid/inline-block`、`position: absolute/fixed`。
- 居中：flex、grid、absolute + transform、margin auto、line-height。

### 选择器、层叠、定位

必会问题：

- 选择器优先级？
- `position` 各值区别？
- `z-index` 为什么失效？
- `display: none`、`visibility: hidden`、`opacity: 0` 区别？

关键词：

- 优先级：内联 > ID > class/属性/伪类 > 标签/伪元素。
- `z-index` 只在定位元素或 flex/grid item 等特定场景生效，并受层叠上下文影响。
- `display:none` 不占位、不渲染；`visibility:hidden` 占位；`opacity:0` 占位且可点击，需配合 pointer-events。

### 响应式与动画

必会问题：

- rem、em、vw、px 区别？
- 响应式布局方案？
- transition 和 animation 区别？
- 为什么用 transform 做动画？

关键词：

- 移动端适配：媒体查询、rem、vw、flex/grid、响应式图片。
- 动画优化：优先 transform/opacity，避免频繁触发 layout。

## 6. Vue 必会

### 基础

必会问题：

- Vue 的响应式原理？
- `computed` 和 `watch` 区别？
- `v-if` 和 `v-show` 区别？
- `data` 为什么是函数？
- `nextTick` 原理？
- `key` 的作用？

答题关键词：

- Vue2：`Object.defineProperty` 劫持 getter/setter，依赖收集和派发更新。
- Vue3：`Proxy` 代理对象，能力更完整，可监听新增/删除属性、数组索引等。
- computed 有缓存，依赖不变不重新计算；watch 适合副作用和异步逻辑。
- `nextTick` 把回调放到 DOM 更新之后，底层优先使用微任务。
- key 帮助 diff 识别节点身份，减少错误复用。

### 生命周期与组件通信

必会问题：

- Vue 生命周期顺序？
- 父子组件生命周期执行顺序？
- 组件通信方式？
- keep-alive 原理和生命周期？

通信方式：

- 父子：props、emit、v-model、ref。
- 跨层级：provide/inject。
- 兄弟/复杂：事件总线、Vuex/Pinia。
- 插槽：普通插槽、具名插槽、作用域插槽。

### 虚拟 DOM 与 Diff

必会问题：

- 为什么需要虚拟 DOM？
- Vue diff 算法？
- 为什么不建议用 index 做 key？

答题关键词：

- 虚拟 DOM 是 JS 对象描述 UI，跨平台、批量更新、降低直接 DOM 操作复杂度。
- Vue2 双端 diff，Vue3 增加静态提升、patch flag、block tree 等编译期优化。
- index 做 key 在插入、删除、排序时会导致节点错误复用。

### Vue2 和 Vue3

必会问题：

- Vue2 和 Vue3 区别？
- Composition API 好处？
- `watch` 和 `watchEffect` 区别？

关键词：

- 响应式从 `defineProperty` 到 `Proxy`。
- 更好的 TS 支持。
- Fragment、Teleport、Suspense。
- 编译优化：静态提升、patch flag、tree shaking。
- Composition API 更适合逻辑复用和大型组件组织。

## 7. React 够用版

如果你主攻 Vue，React 不需要背到非常深，但下面要能讲：

- 类组件和函数组件区别。
- hooks 为什么不能写在条件语句里。
- `useEffect` 和 `useLayoutEffect` 区别。
- React Fiber 解决什么问题。
- React diff 和 key。
- 受控/非受控组件。

> Fiber 把渲染任务拆成可中断的小单元，让 React 可以调度不同优先级的更新，避免长时间同步渲染阻塞主线程。

## 9. 性能优化必会

### 首屏优化

答题框架：

1. 指标：FCP、LCP、TTI/INP、CLS、白屏时间。
2. 资源：压缩、CDN、缓存、HTTP/2、图片优化。
3. JS：拆包、按需加载、tree shaking、减少主线程长任务。
4. CSS：关键 CSS 内联、避免阻塞、减少无用 CSS。
5. 渲染：骨架屏、SSR/SSG、预渲染、懒加载。
6. 数据：接口合并、缓存、并发控制、首屏接口优先级。

项目表达模板：

> 我先通过性能面板和埋点定位瓶颈，区分是资源下载慢、JS 执行慢、接口慢还是渲染慢。然后针对性做拆包、CDN、缓存、图片压缩、首屏组件异步加载、骨架屏和接口并发优化，最后用指标对比验证。

### 图片优化

必会点：

- 选择格式：图标用 SVG，小图可 base64，照片用 JPEG/WebP/AVIF，透明图用 PNG/WebP。
- 响应式图片：按屏幕尺寸加载不同规格。
- 懒加载：`loading="lazy"` 或 IntersectionObserver。
- CDN 裁剪压缩。

### 长列表/虚拟列表

必会点：

- 不一次性渲染全部 DOM，只渲染可视区和缓冲区。
- 固定高度：根据 `scrollTop` 计算 start/end。
- 不定高：缓存高度，动态修正偏移。
- 注意滚动容器、占位高度、item 复用、滚动抖动。

## 10. 前端安全必会

## 12. 项目亮点高频表达

### 复杂表格/BI 报表

可背版本：

> 我负责复杂透视表查看态能力建设。底层使用 VTable 这种 canvas 表格引擎，我在上层封装 BI adapter，把报表 schema 转成 VTable 的行列树、指标、records、样式和交互配置。字段渲染根据类型走 formatter 或自定义 renderer；排序不是前端 sort，而是根据点击的表头路径生成排序参数并重新查询；冻结列是读取业务配置生成 frozenColCount；钻取则根据当前单元格的行列路径组装上下文条件进入下一级报表。

追问点：

- adapter 是业务 schema 到底层组件配置的翻译层。
- canvas 表格性能来自虚拟化和可视区域绘制。
- 冻结列不是 CSS sticky，而是表格引擎分区绘制。

### 仪表盘

可背版本：

> 仪表盘查看态的难点是区块类型很多，比如报表、富文本、Web 区块等。我参与的是统一渲染协议和通用能力接入。每个区块有 type、layout、config、data、permission，主页面只负责布局、权限、复制分享、导出状态，具体内容通过 type 分发到对应 renderer。这样新增区块类型时只新增 renderer，不需要改整个仪表盘。

## 13. 算法最低复习集合

不建议把 100 多道单题逐个背。前端面试常见够用集合：

- 数组/哈希：两数之和、三数之和、合并区间、最长连续序列。
- 字符串/滑动窗口：无重复字符最长子串、最小覆盖子串、有效括号。
- 链表：反转链表、合并链表、环形链表、K 个一组翻转。
- 栈队列：最小栈、每日温度、滑动窗口最大值。
- 二叉树：层序遍历、最大深度、最近公共祖先、路径和。
- 二分：搜索旋转数组、查找左右边界。
- 动态规划：爬楼梯、最长递增子序列、最长公共子序列、零钱兑换。

刷题方法：

- 每类背模板，不背答案。
- 能说复杂度。
- 能讲为什么用这个数据结构。

## 14. 最后 3 天复习顺序

第 1 天：

- JS：原型、闭包、this、异步、事件循环、Promise、手写题。
- 浏览器：渲染、缓存、安全、存储。

第 2 天：

- Vue：响应式、diff、nextTick、组件通信、Vue2/Vue3。
- 网络：HTTP、HTTPS、DNS、TCP、跨域。
- CSS：布局、BFC、Flex、回流重绘。

第 3 天：

- 工程化：Webpack、Vite、Monorepo。
- 性能优化：首屏、包体积、图片、长列表、监控。
- 项目表达：复杂表格、仪表盘、权限、上传、微前端。
- 手写题：防抖节流、深拷贝、Promise.all、bind、new、EventEmitter。

## 15. 不建议重点背的内容

- 所有公司面经逐篇背。
- 粉丝简历、求职模板、HR 模板。
- 小程序支付/发布流程全套细节，除非岗位明确要求小程序。
- 全部设计模式逐个背，只保留发布订阅、策略、代理、单例、适配器的业务例子。
- Web Components、低代码、PDF 预览、Web Worker 深水区，除非你项目里明确写了。
- 过旧 Webpack 优化方案细节，如 HappyPack、DllPlugin，可知道历史背景，不作为主答案。
