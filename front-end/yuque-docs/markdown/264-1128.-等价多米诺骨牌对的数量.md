# 1128. 等价多米诺骨牌对的数量

> 来源：https://www.yuque.com/xiumubai/doc/hkvqcws360ef58h2

1. 题目描述
给你一个由一些多米诺骨牌组成的列表 dominoes。

如果其中某一张多米诺骨牌可以通过旋转 0 度或 180 度得到另一张多米诺骨牌，我们就认为这两张牌是等价的。

形式上，dominoes[i] = [a, b] 和 dominoes[j] = [c, d] 等价的前提是 a==c 且 b==d，或是 a==d 且 b==c。

在 0 <= i < j < dominoes.length 的前提下，找出满足 dominoes[i] 和 dominoes[j] 等价的骨牌对 (i, j) 的数量。

示例：
​JavaScriptRun CodeCopy912输入：dominoes = [[1,2],[2,1],[3,4],[5,6]]输出：1提示：
●1 <= dominoes.length <= 40000
●1 <= dominoes[i][j] <= 9
2. 解题思路
扎个题目中，我们要找的是翻转之后相同数组的对数，所以可以将一组中的两组数字转化为两位数，让大数在前，小数灾后，这样，只要两个数组翻转后能相等，那么他们转化成的两位数也一定是相等。

由于这两个数字都是1-9的数，所以，我们可以初始化一个长度为100的数组，将转化成的两位数作为索引，在数组中进行标记，这样如果有一样的，结果就加一。

复杂度分析:
●时间复杂度：O(N)， 需要遍历一遍数组，其中N是数组的长度
●空间复杂度：O(1)， 因为1 <= dominoes[i][j] <= 9所以键值得组合是有限的常数
3. 代码实现
