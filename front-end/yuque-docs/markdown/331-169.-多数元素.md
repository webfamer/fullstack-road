# 169. 多数元素

> 来源：https://www.yuque.com/xiumubai/doc/vdfpq1tf9mciboxg

1. 题目描述
给定一个大小为 n 的数组，找到其中的多数元素。多数元素是指在数组中出现次数大于 ⌊ n/2 ⌋ 的元素。

你可以假设数组是非空的，并且给定的数组总是存在多数元素。

示例 1:
​912输入: [3,2,3]输出: 3示例 2:
​912输入: [2,2,1,1,1,2,2]输出: 22. 解题思路
对于这道题目，最直接的办法就是将数组中的元素以及其次数保存早map中，然后遍历数组，找出大于次数大于边界值的元素。

该方法的时间复杂度为O(n)，空间复杂度为O(n)。

可以使用双指针求解，具体实现思路如下：
●首先将数组进行排序
●之后定义两个指针i和j，i在左，j在右。首先i不动，j不断向右移动，直到和i对应的元素不相同时停止
●若此时满足j - i大于边界值，就返回结果，如果小于边界值，就将i移动到j的位置，继续向右移动j，重复上一步
●如果j已经到最后一个元素，就返回此时的i对应的值

可以使用摩尔投票算法来解决。摩尔投票算法的基本思想是通过消除不同元素之间的抵消，从而找到出现次数最多的元素。在摩尔投票算法中，我们使用两个变量 count 和 candidate。初始时，将 count 设置为 0，并且 candidate 设置为任意一个元素。然后遍历数组 nums，对于每个元素 num：
●如果 count 为 0，表示当前没有候选元素，将 candidate 设置为当前元素 num。
●如果 num 等于 candidate，则将 count 加 1；否则将 count 减 1。
这样，当遍历完整个数组后，candidate 中存储的就是出现次数超过半数的元素。
