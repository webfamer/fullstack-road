负责项目的搭建，从0到1进行开发并上线；

将代码进行分包处理，按需加载，减小首屏加载时间，提高用户体验。

主要实现功能：
事项申报功能：这是一个动态表单页面，通过后台配置需递交材料项前端调用接口根据后端接口返回的字段生成若干个表单，并从接口返回的类型匹配前端定义好的枚举值设置表单项类型，并在提交时根据字段自定义校验规则。
事项清单功能：遵循react hooks的使用规范，结合taro ScrollView组件封装一个列表组件，包含上拉刷新和下拉加载更多功能，可以根据项目需求自定义刷新和加载的逻辑，更好地适配项目的业务场景。
办件地点查询：有些业务不支持线上需要用户去线下进行办理，使用taro的map组件并根据获取到的办件地点在地图上进行星标，用户点击后调用wx.openLocation Api进行导航。
项目难点：
事项申报页面的动态表单功能，封装一个工具函数将后端返回的数据拼接成表单项，在进行表单校验时由于每一个独立的表单都要绑定一个ref来获取表单项的值，使用useRef持久化一个数组在生成表单的同时使用React.createRef()创建对应的ref并添加特定标识挂在到表单上，在提交表单的时候由于有若干个表单，要按照执行顺序进行验证，form表单的校验是异步操作，使用async/await结合for of完成异步迭代，保证表单校验顺序是正确的。
