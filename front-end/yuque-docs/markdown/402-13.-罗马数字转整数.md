# 13. 罗马数字转整数

> 来源：https://www.yuque.com/xiumubai/doc/isqszcyqgziqbuci

《[传送门](https://leetcode.cn/problems/roman-to-integer/)》
1. 题目描述
罗马数字包含以下七种字符: I， V， X， L，C，D 和 M。
​912345678字符          数值I             1V             5X             10L             50C             100D             500M             1000例如， 罗马数字 2 写做 II ，即为两个并列的 1。12 写做 XII ，即为 X + II 。 27 写做  XXVII, 即为 XX + V + II 。

通常情况下，罗马数字中小的数字在大的数字的右边。但也存在特例，例如 4 不写做 IIII，而是 IV。数字 1 在数字 5 的左边，所表示的数等于大数 5 减小数 1 得到的数值 4 。同样地，数字 9 表示为 IX。这个特殊的规则只适用于以下六种情况：
●I 可以放在 V (5) 和 X (10) 的左边，来表示 4 和 9。
●X 可以放在 L (50) 和 C (100) 的左边，来表示 40 和 90。
●C 可以放在 D (500) 和 M (1000) 的左边，来表示 400 和 900。

给定一个罗马数字，将其转换成整数。输入确保在 1 到 3999 的范围内。

示例 1:

​912输入: "IV"输出: 4示例 3:
​912输入: "IX"输出: 9示例 4:
​9123输入: "LVIII"输出: 58解释: L = 50, V= 5, III = 3.示例 5:

●题目所给测试用例皆符合罗马数字书写规则，不会出现跨位等情况。
●IC 和 IM 这样的例子并不符合题目要求，49 应该写作 XLIX，999 应该写作 CMXCIX 。
●关于罗马数字的详尽书写规则，可以参考 罗马数字 - Mathematics 。
2. 解题思路
这个题目比较简单，就是将每个数字对应的罗马仕用map存储起来，还有几个特殊的值也要保存起来。

随后，对特殊的值进行遍历，如果s中有这几个值，就替换为其对应的数字，将其加入到结果中。

最后遍历s，将剩余罗马符号进行匹配，获取对应的数字，也将其加入到结果中。
3. 代码实现
​JavaScriptRun CodeCopy991234567891011121314151617181920212223242526272829303132333435/** * @param {string} s * @return {number} */var romanToInt = function(s) {    const list = {        I: 1,        V: 5,        X: 10,        L: 50,        C: 100,        D: 500,        M: 1000,    }    const roman  = {        IV:4,        IX:9,        XL:40,        XC:90,        CD:400,        CM:900    }    let result = 0    for(const key in roman){        if(s.includes(key)){            const reg = new RegExp(key)            s = s.replace(reg, '')            result += roman[key]        }    }    for(const i of s){        result += list[i]    }    return result};4.提交结果
