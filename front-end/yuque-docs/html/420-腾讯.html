1typeof和instanceof的区别
typeof和instanceof是JavaScript中用于检测数据类型的操作符，它们有以下区别：
●typeof用于确定一个变量或表达式的数据类型，它返回一个字符串，表示该值的数据类型。常见的返回值包括 "undefined"、"number"、"string"、"boolean"、"object"、"function" 和 "symbol"。例如，typeof 42 返回 "number"，typeof "hello" 返回 "string"。
●instanceof用于检查一个对象是否属于某个特定的构造函数的实例。它检查对象的原型链，如果对象的原型链中出现了指定构造函数的原型，则返回 true，否则返回 false。例如，obj instanceof Array 用于检查 obj 是否为 Array 的实例。
总结：
typeof检测的是值的数据类型，返回一个字符串。
instanceof检测的是对象是否属于某个构造函数的实例，返回一个布尔值。
需要注意的是，typeof null 返回 "object"，这是一个历史遗留问题，因为在JavaScript的早期版本中，null被错误地标识为对象。
2聊一聊dns劫持
1DNS劫持是一种网络攻击手法，它利用对域名系统（DNS）进行篡改来劫持用户的网络流量，导致用户无法正确访问目标网站或被重定向到恶意网站。(输⼊京东被强制跳转到淘宝这就属于dns劫持)。下面是一些关于DNS劫持的讨论：
2DNS解析过程：在正常情况下，当用户输入一个域名进行访问时，计算机会首先向DNS服务器发送查询请求，以获取域名对应的IP地址。然后，计算机使用该IP地址与目标服务器建立连接并获取网页内容。DNS劫持利用了这个过程中的漏洞。
3DNS劫持原理：攻击者可以通过多种方式进行DNS劫持，包括在本地主机、本地网络设备或ISP级别进行篡改。他们会修改DNS响应，将目标域名解析到攻击者控制的恶意IP地址上。这样，当用户尝试访问目标网站时，他们实际上被重定向到攻击者指定的网站。
4影响和风险：DNS劫持可能会导致用户访问受限、个人信息泄露、恶意软件下载等问题。攻击者可以利用劫持的网络流量进行钓鱼攻击、广告注入、恶意重定向等活动，给用户带来损失和安全风险。
5防御和预防：为了防止DNS劫持，可以采取以下措施：
○使用可靠的DNS解析服务，如公共DNS服务器或第三方DNS解析服务。
○定期检查本地网络设备和路由器的配置，确保没有被篡改。
○使用HTTPS协议来保证通信的安全性和完整性，以防止中间人攻击。
○安装和更新防病毒软件、防火墙等安全工具，及时发现和阻止恶意软件。
总结起来，DNS劫持是一种常见的网络攻击手法，它通过篡改DNS解析来劫持用户的网络流量。为了保护自己免受此类攻击的影响，用户需要保持警惕，采取安全措施，并保持系统和软件的更新。
3说一下你对generator的了解？
Generator（生成器）是一种特殊类型的函数，它可以暂停执行并在需要时恢复执行。它是ES6引入的一个重要特性，通过使用function*关键字定义。
以下是对Generator的一些基本理解和特点：
1暂停和恢复执行：Generator函数可以在执行过程中暂停执行，并在需要时恢复执行。在函数体内部，可以使用yield关键字暂停函数执行并产生一个值，然后可以使用next()方法继续执行，并将一个新的值传递给yield表达式。
2迭代器和可迭代对象：Generator函数返回一个迭代器对象，可以通过调用其next()方法逐步遍历生成的值。同时，Generator函数本身也是可迭代的，可以使用for...of循环等语法直接遍历生成的值。
3惰性求值：Generator函数的特点之一是惰性求值，即只有在需要时才生成值。这种特性使得Generator函数在处理大量数据或无限序列时非常有用，可以逐步产生结果，避免一次性计算所有值。
4状态保存：Generator函数在执行过程中可以保存其内部的状态。每次调用next()方法时，函数可以从上一次暂停的位置继续执行，保持局部变量的状态。
5异常处理：Generator函数可以使用try...catch语句捕获异常，并在需要时进行处理。当异常被捕获时，Generator函数会停止执行，并返回一个包含value和done属性的对象。
6Generator函数在实际应用中有许多用途，例如实现惰性计算、异步编程、迭代算法等。通过使用yield表达式，我们可以编写更具可读性和灵活性的代码，实现复杂的流程控制和数据处理。
4说一下macrotask 和 microtask？
在 JavaScript 中，任务分为宏任务（Macrotask）和微任务（Microtask），它们是用来管理异步代码执行的机制。它们有不同的执行优先级和触发时机。
宏任务（Macrotask）是由浏览器提供的任务源，包括以下几种情况：
●setTimeout ，setInterval ，setImmediate 一类的定时事件
● script 脚本的执行
●XMLHttpRequest 和 fetch 请求的回调函数
●requestAnimationFrame 回调函数
●页面渲染后触发的回调函数
宏任务会被放入任务队列中，在主线程空闲时执行，每个宏任务之间都会有一个渲染的间隔。当主线程执行完一个宏任务后，会检查是否有微任务需要执行，然后执行微任务队列中的所有任务，执行完毕后再继续执行下一个宏任务。
微任务（Microtask）是在当前任务执行结束后立即执行的任务，它的优先级高于宏任务。常见的微任务有以下几种：
●Promise 的回调函数（.then() 和 .catch()）
●对 Dom 变化监听的 MutationObserver
●process.nextTick（Node.js 环境）
微任务会被添加到微任务队列中，而不是任务队列。当主线程执行完当前任务后，会立即检查并执行微任务队列中的所有任务，直到微任务队列为空，然后再执行下一个宏任务。
总结：宏任务是整体的、相对较长的任务，而微任务是对当前任务的一种补充，是相对较短的任务，执行时机在当前任务执行结束后立即执行。理解宏任务和微任务的执行顺序和优先级，有助于我们更好地处理异步代码的执行流程和控制。
5Http请求中的keep-alive有了解吗？
HTTP1.0 中默认是在每次请求/应答，客户端和服务器都要新建一个连接，完成之后立即断开连接，这就是短连接。当使用Keep-Alive模式时，Keep-Alive功能使客户端到服务器端的连接持续有效，当出现对服务器的后继请求时，Keep-Alive功能避免了建立或者重新建立连接，这就是长连接。其使用方法如下：
HTTP1.0版本是默认没有Keep-alive的（也就是默认会发送keep-alive），所以要想连接得到保持，必须手动配置发送Connection: keep-alive字段。若想断开keep-alive连接，需发送Connection:close字段；
HTTP1.1规定了默认保持长连接，数据传输完成了保持TCP连接不断开，等待在同域名下继续用这个通道传输数据。如果需要关闭，需要客户端发送Connection：close首部字段。
Keep-Alive的建立过程：
1客户端向服务器在发送请求报文同时在首部添加发送Connection字段
2服务器收到请求并处理 Connection字段
3服务器回送Connection:Keep-Alive字段给客户端
4客户端接收到Connection字段
5Keep-Alive连接建立成功
6服务端自动断开过程（也就是没有keep-alive）：
7客户端向服务器只是发送内容报文（不包含Connection字段）
8服务器收到请求并处理
9服务器返回客户端请求的资源并关闭连接
10客户端接收资源，发现没有Connection字段，断开连接
11客户端请求断开连接过程：
12客户端向服务器发送Connection:close字段
13服务器收到请求并处理connection字段
14服务器回送响应资源并断开连接
15客户端接收资源并断开连接
开启Keep-Alive的优点：
1较少的CPU和内存的使⽤（由于同时打开的连接的减少了）；
2允许请求和应答的HTTP管线化；
3降低拥塞控制 （TCP连接减少了）；
4减少了后续请求的延迟（⽆需再进⾏握⼿）；
5报告错误⽆需关闭TCP连；
开启Keep-Alive的缺点：
1长时间的Tcp连接容易导致系统资源无效占用，浪费系统资源
6请描述一下DNS解析的具体过程？
DNS服务器解析域名的过程：
1首先会在浏览器的缓存中查找对应的IP地址，如果查找到直接返回，若找不到继续下一步
2将请求发送给本地DNS服务器，在本地域名服务器缓存中查询，如果查找到，就直接将查找结果返回，若找不到继续下一步
3本地DNS服务器向根域名服务器发送请求，根域名服务器会返回一个所查询域的顶级域名服务器地址
4本地DNS服务器向顶级域名服务器发送请求，接受请求的服务器查询自己的缓存，如果有记录，就返回查询结果，如果没有就返回相关的下一级的权威域名服务器的地址
5本地DNS服务器向权威域名服务器发送请求，域名服务器返回对应的结果
6本地DNS服务器将返回结果保存在缓存中，便于下次使用
7本地DNS服务器将返回结果返回给浏览器
比如要查询 [www.baidu.com](http://www.baidu.com/) 的 IP 地址，首先会在浏览器的缓存中查找是否有该域名的缓存，如果不存在就将请求发送到本地的 DNS 服务器中，本地DNS服务器会判断是否存在该域名的缓存，如果不存在，则向根域名服务器发送一个请求，根域名服务器返回负责 .com 的顶级域名服务器的 IP 地址的列表。然后本地 DNS 服务器再向其中一个负责 .com 的顶级域名服务器发送一个请求，负责 .com 的顶级域名服务器返回负责 .baidu 的权威域名服务器的 IP 地址列表。然后本地 DNS 服务器再向其中一个权威域名服务器发送一个请求，最后权威域名服务器返回一个对应的主机名的 IP 地址列表。
7聊一聊队头阻塞问题，如何解决？
队头阻塞问题是指在一些网络通信或异步操作中，由于前面的任务耗时较长，导致后续任务被阻塞，无法及时执行的情况。这种问题通常出现在使用单线程的事件循环模型中，例如在JavaScript中的异步回调或Promise链式调用中。

队头阻塞可能导致用户界面的卡顿、延迟响应或阻塞其他关键任务的执行。为了解决队头阻塞问题，可以采取以下几种策略：
1分割任务：将长时间运行的任务切分为更小的任务单元，使其能够分散在事件循环的不同迭代中执行。这样可以避免某个任务长时间占用事件循环，阻塞后续任务的执行。
2使用Web Workers：Web Workers 是在后台运行的独立线程，可以执行耗时的计算任务，而不会阻塞主线程。将一些计算密集型任务转移到Web Workers中，可以减少对主线程的阻塞。
3异步编程技术：使用异步编程技术，如Promise、async/await、回调函数等，可以将耗时的任务转换为非阻塞的异步操作。这样可以让事件循环在等待异步操作完成期间继续处理其他任务，避免队头阻塞。
4使用定时器：使用定时器（setTimeout、setInterval）将任务延迟执行，可以让事件循环在每个迭代中有时间处理其他任务。将长时间运行的任务拆分为多个阶段，每个阶段使用定时器延迟执行，以避免阻塞事件循环。
5使用WebAssembly：WebAssembly 是一种高性能的二进制格式，可以在浏览器中运行更接近原生代码的程序。将一些性能敏感的任务使用WebAssembly编写，并在独立的线程中执行，可以减少对主线程的阻塞。
总的来说，解决队头阻塞问题的关键是将耗时的任务切分为更小的任务单元，让事件循环在每个迭代中有机会处理其他任务。通过合理地使用异步编程技术、Web Workers、定时器和WebAssembly等工具，可以优化性能并提高用户体验。
8你觉得virtual dom 比真实的dom更快的吗
在某些情况下，Virtual DOM 可以比真实的 DOM 操作更快。这是因为 Virtual DOM 通过在内存中维护一个虚拟的 DOM 树，将对真实 DOM 的操作转换为对虚拟 DOM 的操作，然后通过 diff 算法找出需要更新的部分，最后只更新这些变化的部分到真实的 DOM。
虽然 Virtual DOM 需要进行额外的计算来比较虚拟 DOM 和真实 DOM 的差异，但在多次更新中，它可以批量处理这些变化，从而减少对真实 DOM 的访问和操作次数。这样可以减少浏览器的重排和重绘，提高页面的性能和响应速度。
另外，Virtual DOM 还可以实现批量更新和异步更新，它可以将多个更新操作合并为一次更新，并在合适的时机进行更新，减少了不必要的中间状态的更新，提高了性能。
然而，在某些简单的场景下，直接操作真实的 DOM 可能更快。当更新的操作很少或者只是局部的改变时，直接操作真实 DOM 可能比 Virtual DOM 更高效，因为它省去了 Virtual DOM 的比较和计算的开销。
综上所述，虽然 Virtual DOM 在某些情况下可以提供更好的性能，但在简单的场景下直接操作真实 DOM 可能更快。在实际开发中，需要根据具体情况选择适合的技术来平衡性能和开发效率。
9项目怎么做的移动端适配？
移动端适配是保证网页在不同设备上显示和交互效果一致的重要工作。下面是一些常见的移动端适配方法：
1媒体查询（Media Queries）：使用CSS的@media规则根据屏幕尺寸、分辨率等特性来调整布局和样式。可以通过设置不同的CSS样式表或使用响应式的CSS框架来适配不同屏幕尺寸。
2视口（Viewport）设置：通过设置标签的viewport属性来控制页面的缩放行为和视口大小。常用的设置包括设置视口宽度、禁用缩放、设置初始缩放比例等。
3REM布局：使用相对单位REM来设置元素的尺寸，基于根元素的字体大小进行缩放。可以根据屏幕宽度动态设置根元素的字体大小，从而实现页面元素的自适应布局。
4Flexbox布局：使用CSS的Flexbox布局模型来创建灵活的布局结构，能够适应不同屏幕尺寸和方向的变化。通过设置弹性盒子的属性和属性值，可以实现响应式的布局和对齐方式。
5图片适配：使用响应式的图片或使用CSS的background-image属性来适配不同屏幕尺寸。可以通过设置不同尺寸的图片或使用CSS的background-size属性来控制图片的缩放和裁剪。
6手势事件：使用JavaScript监听并处理移动设备的触摸事件，实现移动端特定的交互效果，例如滑动、缩放、轮播等。
7使用移动端框架：使用移动端开发框架，如React Native、Ionic、Flutter等，这些框架提供了针对移动端开发的组件和样式库，可以简化适配工作。
在实际开发中，根据项目需求和目标设备的特性，可以选择合适的移动端适配方法或结合多种方法来实现适配效果。同时，通过使用浏览器的开发者工具和在真实设备上进行测试，可以进行调试和优化，确保移动端适配的效果和体验。
10算法两道：爬楼梯和背包找零问题
1爬楼梯问题：
题目描述：假设你正在爬楼梯，每次只能爬一步或两步。编写一个函数，给定楼梯的总层数 n，计算你可以爬到楼顶的不同方法总数。

解题思路：这是一个经典的动态规划问题。定义一个长度为 n+1 的数组 dp，其中 dp[i] 表示爬到第 i 层楼梯的不同方法总数。根据题目要求，dp[1] = 1，dp[2] = 2，对于第 i 层楼梯，可以从第 i-1 层跨一步到达，或者从第 i-2 层跨两步到达，因此有递推关系 dp[i] = dp[i-1] + dp[i-2]。最终返回 dp[n] 即可。
2背包找零问题：
题目描述：假设你是一位收银员，手上有不同面额的硬币，现在需要找零 money 元。编写一个函数，计算需要找零的最少硬币个数。

解题思路：这也是一个动态规划问题。定义一个长度为 money+1 的数组 dp，其中 dp[i] 表示找零 i 元所需的最少硬币个数。初始化 dp 数组为无穷大，dp[0] = 0。对于每个金额 i，遍历所有的硬币面额 coins[j]，如果当前硬币面额 coins[j] 小于等于金额 i，并且 dp[i-coins[j]] + 1 的值小于 dp[i]，则更新 dp[i] = dp[i-coins[j]] + 1。最终返回 dp[money]。
