基础
Base64 是一种编码方式，用于将二进制数据转换为可打印的 ASCII 字符串，它的原理是将原始数据按照固定的规则进行编码和解码，以便在文本协议中传输或存储。
Base64 编码具有以下特点和问题：
1增加了数据大小：由于 Base64 将二进制数据编码为 ASCII 字符串，编码后的数据大小通常会比原始数据大约 33%。这是因为每 3 字节的数据会转换为 4 个 Base64 字符，导致数据膨胀。因此，Base64 编码会增加数据的大小，可能会对存储和传输造成一些额外的开销。
2字符集限制：Base64 编码后的字符串只包含 ASCII 字符，因此只能用于支持 ASCII 字符集的环境中。如果需要在非 ASCII 环境中传输或存储数据，可能需要使用其他编码方式。
3不可读性：Base64 编码后的字符串包含大小写字母、数字和特殊字符，不具备直观的可读性。虽然它是可打印的 ASCII 字符串，但不适合直接阅读。
Base64 编码适合在以下情况下使用：
1传输二进制数据：在文本协议中，如果需要传输二进制数据（如图像、音频、视频等），可以使用 Base64 编码将其转换为字符串，以便在文本协议中传输或嵌入(在 HTTP1.x 场景下，还可减少资源请求数，提升网络性能)；
2URL 参数传递：在 URL 中传递参数时，某些字符可能会被解析器解释为特殊字符或导致编码问题。使用 Base64 编码可以将参数转换为安全的 ASCII 字符串，避免这些问题。
3存储数据：在某些场景下，需要将二进制数据存储为文本格式，如数据库中的文本字段。使用 Base64 编码可以将二进制数据转换为字符串，便于存储和检索。
需要注意的是，Base64 编码只是一种编码方式，并非加密算法。它不提供数据的保密性，只是将数据转换为不可直接读取的形式。如果需要保护数据的机密性，应使用适当的加密算法。
进阶
为什么
Base64 的基本功能是以 ASCII 字符串表达二进制数据，因此其主要应用于“无法”或者“不希望”使用二进制编码的场景，一个非常经典的应用就是在 HTML 中嵌入多媒体资源(例如图片)。举例来说，HTML 是一种文本协议，自然是无法写入二进制数据的，设想这样一段代码：

进一步梳理一下，常见的 Base64 应用场景有：
1文本传输：在许多协议和技术中，只支持文本数据的传输(例如上述的 HTML)。但有时需要传输或存储二进制数据，如图像、音频、视频等。Base64 编码提供了一种将二进制数据转换为文本形式的方法，以便在这些协议和技术中进行传输。通过将二进制数据编码为文本，可以确保数据能够准确地传输，而不会受到特殊字符或编码方式的限制。
2数据表示：在某些情况下，需要使用文本表示的方式来表示二进制数据，例如在配置文件、数据库字段(当然，很多数据库其实支持二进制类型)或文本文档中存储二进制数据。Base64 编码提供了一种简单的方式，将二进制数据转换为文本形式进行存储和表示。
3URL 和文件名安全性：在将数据作为 URL 参数传递或作为文件名使用时，可能需要确保数据的安全性和兼容性。Base64 编码可以确保数据只包含安全的字符集（字母、数字和少量特殊字符），避免了特殊字符对 URL 解析或文件系统的影响。
4加密和签名：在一些加密和签名算法中，要求输入的数据为文本形式。Base64 编码提供了一种方便的方法，将二进制数据转换为文本形式进行处理。例如，在使用公钥加密算法时，可以将加密后的数据进行 Base64 编码后再进行传输。
编码过程
Base64 字符表由 64 个字符组成，包括大小写字母、数字和两个特殊字符（通常是 "+" 和 "/"，或者 "-" 和 "_"）。Base64 编码步骤大致可总结如下：
1将原始数据划分为连续的 3 字节组（共 8bit*3 = 24 位）；
2将每个 3 字节组拆分为四个 6 位的片段；

1根据 Base64 字符表将这些 6 位片段映射为相应的 ASCII 字符。

1如果原始数据长度不是 3 字节的倍数，则填充 0 直到为 3 的倍数(最后可能会有一个或两个填充字符 "=")
需要注意的点
1Base64 编码规则决定了，编码结果必然比原始数据大 1/3，因此在 web 场景下，可能会增加网络负担；
2Base64 只是一种内容编码协议，编解码过程都不存在任何加解密步骤，因此不存在“提升安全性”一说，任何人都可以通过完全相同的反序列算法获取原文，效果与 JSON 协议无异；
