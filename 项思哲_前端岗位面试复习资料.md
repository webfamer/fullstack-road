## 4. 高频项目讲法

### 4.1 BI 报表分析平台

一句话：

> 这是一个面向业务自助分析的 BI 报表平台，用户可以通过字段配置和拖拽生成报表，并在仪表盘里查看、分享、导出和移动端访问。

面试官可能问：

1. 复杂表格是怎么设计的？
2. 图表/表格组件体系怎么抽象？
3. 报表页面首屏慢怎么优化？
4. 字段类型和渲染配置如何扩展？
5. 仪表盘多类型内容如何统一渲染？

建议回答框架：

```text
业务问题：报表类型多、字段类型多、查看态交互复杂，多个业务线都要复用。
技术方案：把数据处理、字段解析、渲染配置、图表/表格渲染器拆开，通过 Storybook 管理和验证组件状态，再沉淀 npm 包复用。
优化动作：图片 CDN 化、骨架屏、组件异步加载、按需渲染，减少白屏等待。
结果：核心报表查看、仪表盘、导出分享、移动端适配都能复用统一能力，减少重复开发。
```

追问准备：

- 表格能力：字段类型渲染、列宽自适应、排序、冻结列、钻取。
- 组件抽象边界：不要让组件直接依赖业务接口，组件只关心数据、字段 Schema、渲染配置、事件回调。
- Storybook 价值：沉淀各种状态样例，便于联调、验收、回归、文档化。

### 4.2 BI 智能问数助手

一句话：

> 这是面向 BI 自助分析的自然语言问数系统，用户输入中文问题后，系统通过元数据检索和 Agent 工作流生成 SQL、校验修正、执行查询，并通过 SSE 流式返回过程和结果。

面试官可能问：

1. 为什么用 SSE，不用 WebSocket？
2. LangGraph 流程怎么拆节点？
3. 如何降低大模型选错表、选错字段的问题？
4. SQL 生成不稳定怎么处理？
5. 前端如何展示流式步骤和中断？

建议回答框架：

```text
业务问题：业务人员不会写 SQL，但需要查询销售额、订单数、品类、区域等指标。
技术方案：用 LangGraph 拆成关键词抽取、字段召回、指标召回、字段值召回、上下文补全、SQL 生成、SQL 校验修正、SQL 执行等节点。
数据基础：表结构、字段描述、字段别名、指标口径放 MySQL/Qdrant/Elasticsearch，语义检索和全文检索结合。
前端交互：SSE 返回执行进度、SQL、结果表格和异常信息，用户可以看到 Agent 正在做什么。
风险控制：执行前增加 SQL 校验和修正，避免错误 SQL 直接查询数据源。
```

SSE 对比 WebSocket：

- SSE：浏览器原生 EventSource，服务端到客户端单向推送，适合 AI 流式输出、进度流、通知流。
- WebSocket：全双工，适合实时协作、在线聊天、游戏、双向高频交互。
- 本项目问数流程主要是“用户发起一次问题，服务端持续返回步骤和结果”，所以 SSE 更轻。

RBAC 回答要点：

- 用户、角色、菜单/权限点三类核心模型。
- 登录后签发 JWT，后端 Guard 校验 token。
- 根据用户角色查询菜单树和权限码。
- 前端根据菜单树生成动态路由，根据权限码控制按钮或操作入口。
- 后端接口仍需校验权限，前端隐藏只做体验优化。

- 先看接口耗时，定位是应用逻辑慢还是数据库慢。
- 打印 SQL 和参数，查看执行计划。
- 常见优化：加索引、避免函数包裹索引字段、减少不必要 join、分页、只查必要字段、按时间范围过滤。
- 对报文/告警这类大表，重点检查时间字段、设备编号、告警类型等查询条件的组合索引。

## 5. React 复习重点

### 5.1 Hooks 核心问题

useState：

- 状态更新是异步批处理。
- 对依赖旧值的更新使用函数式写法：`setCount(prev => prev + 1)`。
- 对复杂对象要保持不可变更新。

useEffect：

- 处理副作用：请求、订阅、定时器、DOM 事件。
- 依赖数组决定执行时机。
- 返回 cleanup 清理副作用，尤其是 WebSocket、EventSource、setInterval、图表实例。

useMemo：

- 缓存计算结果，适合表格列配置、过滤结果、复杂数据转换。
- 不是所有计算都需要缓存，计算成本低时可能没有收益。

useCallback：

- 缓存函数引用，常与 React.memo 子组件配合。
- 适合传给深层子组件、列表项、事件配置。

useRef：

- 保存不触发渲染的可变值。
- 常用于保存 WebSocket/EventSource 实例、ECharts 实例、上一次值、DOM 节点。

### 5.2 React.memo / useMemo / useCallback 怎么讲

核心原则：

> 优化的目标不是“用了 memo”，而是减少无意义重渲染，同时避免引入过度缓存和依赖错误。

典型场景：

- 报表表格列配置较复杂，用 useMemo 固定 columns。
- 子组件较重，用 React.memo 包裹。
- 父组件传给子组件的回调用 useCallback 保持引用稳定。
- 高频 WebSocket 数据进入页面时，先缓冲/节流，再更新状态。

常见坑：

- 依赖数组漏依赖导致闭包旧值。
- 传入对象/数组字面量导致 memo 失效。
- React.memo 浅比较，复杂对象引用变化仍会重渲染。
- 过度使用 useMemo/useCallback 会增加维护成本。

### 5.3 权限路由和动态路由

回答结构：

```text
登录 -> 获取 token -> 获取用户信息/菜单/权限码 -> 生成路由表 -> 渲染 Layout 和菜单 -> 按权限控制页面与按钮 -> 后端接口二次校验
```

关键点：

- 前端路由权限只是体验层，不能作为安全边界。
- 页面级权限：控制路由是否注册或是否可访问。
- 菜单权限：控制侧边栏展示。
- 按钮权限：控制新增、编辑、删除、导出等操作。
- 刷新恢复：从 token 重新拉用户信息和菜单。
- 403：已登录但无权限；401：未登录或 token 失效。

### 5.4 虚拟列表

原理：

- 页面只渲染可视区域附近的数据。
- 通过容器总高度撑开滚动条。
- 根据 scrollTop 计算 startIndex/endIndex。
- 用 transform 或 padding 把可视项移动到正确位置。

适用场景：

- 报文列表、日志列表、消息流、超大表格。

注意点：

- 固定高度实现简单。
- 动态高度需要缓存每项高度，处理滚动位置修正。
- 表格虚拟滚动还要考虑冻结列、横向滚动、表头同步。

### 5.5 首屏优化和包体积优化

你简历里可以讲的动作：

- 图片 CDN 化。
- 骨架屏。
- 组件异步加载。
- 移动端适配。

可以补充的系统化方法：

## 6. Node.js 复习重点

### 6.1 Node 异步模型和事件循环

必须能讲清楚：

- Node.js 单线程指的是 JS 主线程，底层 I/O 依赖 libuv 线程池和操作系统异步能力。
- 事件循环负责调度宏任务、微任务和 I/O 回调。
- Promise.then / queueMicrotask 属于微任务。
- setTimeout、setInterval、setImmediate、I/O callback 属于不同阶段的任务。
- CPU 密集型任务会阻塞事件循环，应该拆分、用 worker_threads、队列或外部服务处理。

常见面试问法：

1. 为什么 Node 适合 I/O 密集型？
2. Promise、setTimeout、setImmediate 执行顺序？
3. 如何判断事件循环被阻塞？
4. 大文件上传为什么不能一次读入内存？

### 6.2 NestJS 项目结构

结合电力设备平台讲：

- Module：按业务域拆分，如 AuthModule、UserModule、DeviceModule、PacketModule、AlarmModule。
- Controller：接收 HTTP 请求，做参数校验和路由分发。
- Service：承载业务逻辑。
- Guard：认证和权限校验，例如 JwtAuthGuard、PermissionGuard。
- Interceptor：统一响应、日志、耗时统计。
- Pipe：参数校验和转换。
- Exception Filter：统一异常格式。

### 6.3 BFF 架构

BFF 是 Backend For Frontend，核心是为前端体验定制后端接口。

适合场景：

- 前端一个页面需要聚合多个后端接口。
- 移动端和 PC 端字段不同。
- 需要统一鉴权、错误格式、字段裁剪、数据转换。
- 后端服务比较多，前端不想直接感知复杂服务边界。

你可用的项目映射：

- BI 问数助手的 FastAPI 接口：封装 Agent 流程，对前端只暴露问数流。
- 电力设备平台 NestJS：给后台管理前端提供设备、报文、告警、权限等接口。

### 6.4 网关服务

回答重点：

- 统一入口：转发到不同后端服务。
- 认证鉴权：校验 token、权限、签名。
- 限流熔断：保护下游服务。
- 日志追踪：记录请求链路、耗时、错误。
- 协议转换：HTTP、WebSocket、内部 RPC。

Nginx 和应用网关区别：

- Nginx 更偏反向代理、静态资源、负载均衡。
- 应用网关更偏业务鉴权、路由策略、限流、审计和协议适配。

### 6.5 SSE / WebSocket

SSE 示例回答：

```text
SSE 基于 HTTP 长连接，服务端持续以 text/event-stream 返回事件。前端用 EventSource 或 fetch 流式读取。它适合问数进度、AI 输出、通知这类服务端单向推送场景。
```

WebSocket 示例回答：

```text
WebSocket 是全双工长连接，适合双方都要主动发送消息的场景，比如训练驾驶舱高频数据、在线协作、实时聊天。
```

清理逻辑：

- 组件卸载时关闭 EventSource/WebSocket。
- 清理定时器、重试任务、事件监听。
- 避免连接对象进入 React state，通常放 useRef。

### 6.6 文件上传与流处理

虽然简历没有强项目体现，但 JD 写了，建议重点补。

普通上传：

- 前端用 FormData。
- Node 端用 multer / busboy 解析 multipart。
- 校验文件类型、大小、权限。
- 保存到本地、对象存储或转发到文件服务。

大文件上传：

- 前端切片，计算 hash。
- 服务端检查已上传分片。
- 并发上传分片。
- 上传完成后服务端合并。
- 支持秒传和断点续传。

Node stream：

- 不把大文件一次性读入内存。
- 使用 readable stream -> transform stream -> writable stream。
- 通过 backpressure 控制读写速度。

面试回答示例：

```text
如果是大文件上传，我会避免一次性读取完整文件。前端做分片和 hash，服务端记录分片状态，分片到齐后用 stream 合并。这样可以降低内存占用，也能支持断点续传和失败重试。
```

## 7. MySQL / Redis / 部署复习重点

### 7.1 MySQL

结合设备平台讲：

- 表设计：设备表、报文表、告警规则表、告警记录表、用户表、角色表、菜单表。
- 索引：设备编号、时间、告警类型、状态字段。
- 慢 SQL：查看执行计划，确认是否走索引。
- 分页：大表避免深分页，可用基于时间/ID 的游标分页。
- 只查必要字段，避免 `select *`。

### 7.2 Redis

可讲场景：

- 登录 token/session 缓存。
- 权限菜单缓存。
- 高频配置缓存。
- 设备最新状态缓存。
- 分布式锁或简单限流。

注意：

- 缓存要考虑过期时间。
- 更新数据库后要处理缓存失效。
- 热点 key、缓存穿透、缓存击穿、缓存雪崩要知道概念。

### 7.3 Docker / Linux / Nginx

前端部署：

- build 产物生成静态文件。
- Nginx 托管 dist。
- 配置 gzip、缓存、history 路由 fallback。
- API 通过反向代理转发到后端。

Node 部署：

- Dockerfile 构建镜像。
- 配置环境变量。
- 容器日志查看。
- 健康检查和重启策略。

线上排查：

- 看 Nginx access/error log。
- 看应用日志。
- 看接口状态码、耗时、错误堆栈。
- 用 curl 验证接口连通性。
- 确认端口、域名、证书、反向代理路径。

## 8. 面试题库和回答提纲

### React

1. React Hooks 为什么不能写在条件语句里？
   - Hooks 依赖调用顺序保存状态，条件调用会破坏顺序。

2. useEffect 和 useLayoutEffect 区别？
   - useEffect 在浏览器绘制后执行，不阻塞渲染。
   - useLayoutEffect 在 DOM 更新后、绘制前同步执行，适合测量布局，但可能阻塞绘制。

3. Context 会导致什么性能问题？
   - Provider value 变化会让消费方重渲染。
   - 可拆分 Context、稳定 value 引用、使用 selector 方案。

4. React key 的作用？
   - 帮助 React 识别列表项身份，减少错误复用。
   - 不稳定 key 会导致状态错乱和额外渲染。

5. 如何优化一个很慢的报表页面？
   - 先定位：Network、Performance、React Profiler。
   - 再处理：接口并行/缓存、路由懒加载、图表按需加载、表格虚拟滚动、memo、图片 CDN、骨架屏。

### Node.js

1. Node 如何处理并发？
   - 单 JS 主线程 + 事件循环 + 异步 I/O + libuv 线程池。

2. Express/Koa/NestJS 区别？
   - Express 简单直接，生态成熟。
   - Koa 更轻，洋葱模型，中间件体验好。
   - NestJS 工程化强，模块、依赖注入、装饰器、Guard/Interceptor 适合中大型项目。

3. JWT 鉴权流程？
   - 登录校验账号密码。
   - 服务端签发 token。
   - 前端请求带 Authorization。
   - 后端 Guard 校验 token 和权限。
   - token 过期返回 401。

4. 如何做接口统一异常？
   - 全局 Exception Filter。
   - 统一错误码、message、requestId。
   - 日志记录堆栈和上下文。

5. 如何避免 Node 服务内存泄漏？
   - 避免全局数组/Map 无限增长。
   - 清理定时器、连接、监听器。
   - 大文件用 stream。
   - 用 heap snapshot、process memory、日志监控定位。

### 实时通信

1. SSE 和 WebSocket 怎么选？
   - 单向服务端推送用 SSE。
   - 双向实时通信用 WebSocket。

2. WebSocket 断线怎么处理？
   - 心跳检测。
   - 指数退避重连。
   - 页面卸载关闭连接。
   - 服务端识别连接状态。

3. 高频数据如何防止 React 页面卡顿？
   - 数据缓冲。
   - 节流/批量更新。
   - 图表增量更新。
   - 避免每条消息都 setState。
   - Web Worker 处理重计算。

### 权限系统

1. 权限路由怎么做？
   - 登录后拉菜单和权限码。
   - 动态生成路由。
   - 前端控制展示，后端接口兜底。

2. RBAC 表怎么设计？
   - user、role、permission/menu、user_role、role_permission。

3. 按钮权限怎么控制？
   - 后端返回权限码。
   - 前端封装权限组件或 hook。
   - 后端接口仍校验。

### 性能优化

1. 首屏白屏怎么排查？
   - 看资源加载、JS 执行、接口耗时、渲染耗时。
   - 用 Performance、Lighthouse、Network、React Profiler。

2. 包体积怎么优化？
   - 路由懒加载。
   - 第三方库按需引入。
   - Tree shaking。
   - 拆分 vendor。
   - 移除重复依赖。

3. 长列表怎么优化？
   - 虚拟列表。
   - 分页/游标分页。
   - 减少 DOM 节点。
   - 固定行高优先，动态行高要缓存高度。
