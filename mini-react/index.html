Mini React / Fiber / 双树

最小 React 实现：把面试里那套流程讲顺

这个 demo 保留 React 最关键的几件事：虚拟 DOM 描述 UI、Fiber 把递归改成可中断链表遍历、current/workInProgress 双树对比、reconcile 产出 effect、commit 一次性改真实 DOM。

运行日志

每次点击按钮都会看到：创建新虚拟树 -> 调度 workLoop -> 生成 workInProgress Fiber 树 -> reconcile 打标记 -> commit 提交到 DOM。 


        

      


      

        
面试时建议这样讲

        

          

            1. 虚拟 DOM：
            它不是为了“更快”，核心是先把 UI 变成 JS 对象树，后面才能做 diff、调度、跨平台。
          

          

            2. Fiber：
            React 以前是递归同步更新，一旦开始很难停。Fiber 把每个节点变成一个工作单元，可以拆片执行。
          

          

            3. 双树：
            current 树代表当前屏幕，workInProgress 树代表正在计算的新结果，两棵树通过
            alternate 互相指向。
          

          

            4. render 阶段：
            只算，不改 DOM。主要做 reconcile，决定节点是新增、更新还是删除。
          

          

            5. commit 阶段：
            不能中断，把 effect 一次性提交到真实 DOM，保证界面一致性。
